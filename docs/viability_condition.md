## Core Theorem

> **The Viability Condition: Ceff(t) > E(t)**
>
> A recursive system that trains on its own outputs remains semantically viable
> only when the rate of externally-verified correction exceeds the rate of
> internally-generated error at all times.

Where:
- **Ceff(t)** = effective corrective bandwidth (verified human ground-truth
  inputs per unit time)
- **E(t)** = error accumulation rate (the rate at which the system's
  representations diverge from verified ground truth)

Violation of this condition causes **informational autophagy**: the system
begins consuming its own semantic grounding, progressively substituting
internally-generated tokens for externally-verified signals. The result is
representation collapse — statistically smooth output that is increasingly
decoupled from the phenomena it claims to describe.

## Theoretical Foundation

### The recursion problem

Modern AI systems are trained on data that is increasingly generated by prior
AI systems. Unlike biological recursion (where error-correction is built into
the cellular machinery), artificial recursion has no intrinsic correction
mechanism. Every synthetic token that enters the training corpus is a potential
error that propagates forward.

The question is not whether synthetic data causes degradation — under any
finite error rate it does — but whether the **rate** of correction is sufficient
to prevent runaway accumulation.

### Formal statement

Let S(t) be the semantic grounding of the system at time t. Then:

```
dS/dt = Ceff(t) - E(t)
```

If `Ceff(t) > E(t)` for all t, then S(t) is non-decreasing (semantic grounding
is preserved or improved).

If `Ceff(t) < E(t)` for any interval, S(t) degrades over that interval —
informational autophagy is occurring.

The condition is **rate-based**, not threshold-based. A model is not "safe" or
"unsafe"; it is operating with a specific ratio of Ceff/E. That ratio must be
monitored and maintained.

### The scaling implication

> "Scaling with synthetic data is viable only when paired with commensurate
> verification infrastructure."

This is not an abstract claim. It means that for every N synthetic tokens added
to a training corpus, the verification infrastructure (human reviewers,
grounding interviews, consensus mechanisms) must scale proportionally. Without
this, the Ceff/E ratio decreases as synthetic data scales, making informational
autophagy increasingly likely.

## Temporal Signature (Falsifiable Prediction)

The Viability Condition generates a falsifiable empirical prediction:

> **Out-of-distribution (OOD) accuracy degrades BEFORE validation perplexity rises.**

This ordering is the temporal signature of informational autophagy:

1. The model's representations first become internally consistent but externally
   decoupled (OOD accuracy falls — the model fails on genuinely novel inputs)
2. Only later does the distribution shift become visible in validation loss
   (because validation data itself may contain synthetic-origin text)

This signature is **detectable in advance** if OOD benchmarks are tracked
alongside standard validation metrics. The HAIC improvement pipeline monitors
both SGT (Semantic Grounding Test — an OOD proxy) and standard loss curves,
specifically to catch this leading indicator.

## Prism Metrics as E(t) Proxies

The PRISM geometry metrics computed by `outlier_geometry()` are operationalisations
of E(t) at the representation level:

| Metric | E(t) proxy | Mechanism |
|---|---|---|
| `outlier_ratio` | Magnitude of representation drift | A dominant "massive activation" signals collapse toward a degenerate attractor |
| `activation_kurtosis` | Volatility / instability | Heavy-tailed distributions indicate representations under pressure from competing attractors |
| `cardinal_proximity` | Systematic quantization error | Axis-aligned activations incur systematic precision loss at inference, compounding error with each forward pass |
| `quantization_hostility` | Aggregate E(t) | Composite score; >0.7 indicates the layer is operating above the threshold where low-bit inference remains viable |

**Interpretation:** A model with `quantization_hostility = 0.91` (e.g. Gemma 4
E2B, `outlier_ratio=83.2`, `kurtosis=1009.5`) is operating with E(t) far above
what most verification infrastructure can match. Maintaining Ceff(t) > E(t) for
this model requires exceptionally high-bandwidth grounding — or the deployment
must be restricted to domains with very low error propagation.

## Maestro as the Ceff(t) Mechanism

The Maestro gateway is the physical implementation of the correction loop. Each
interview session processed through it represents one quantum of Ceff:

```
Human participant
  → HumanCheck (Altcha PoW — proves human, not bot)
  → ConsentGate (5-layer consent — ensures the correction is authorized)
  → Interview (grounding dialogue — the correction event itself)
  → Session lattice (SHA-256 Merkle DAG — makes the correction verifiable)
  → improvement_pipeline.py (integrates correction into next model version)
```

The **consent gate** is not incidental — it is load-bearing for the Viability
Condition. An unconsented or coerced "correction" may introduce its own error
vector. The one-way consent gate ensures that Ceff only receives signals that
are:
1. Voluntary (not coerced — preserving epistemic authenticity)
2. Verified (PoW proves human — preventing synthetic injection into Ceff)
3. Auditable (Merkle receipt — allowing future verification that Ceff was
   computed correctly)

### Ceff estimation formula

```python
ceff_per_day = (

    * avg_turns_per_session
```

## `check_viability_condition` — Reference Implementation

```python
import math

def check_viability_condition(
    model_id: str,
    deployment_context: str,
    error_rate_estimate: float,          # E(t) in corrections-equivalent/day
    verification_bandwidth_estimate: float,  # Ceff(t) in corrections/day
    synthetic_data_ratio: float          # [0, 1] — fraction of training data synthetic
) -> dict:
    """
    Evaluate the Viability Condition Ceff(t) > E(t) for a model/deployment.

    Args:
        model_id: identifier for PRISM arena lookup
        deployment_context: free-text description of deployment domain
        error_rate_estimate: estimated error accumulation rate in
            corrections-equivalent/day. Can be derived from Prism metrics:
            E(t) ≈ quantization_hostility * deployment_scale_factor
        verification_bandwidth_estimate: estimated corrections/day entering
            the grounding pipeline via Maestro
        synthetic_data_ratio: fraction [0,1] of training data that is
            synthetic-origin (reduces effective Ceff)

    Returns:
        {
            viability_satisfied: bool,
            ceff_vs_e_ratio: float,
            autophagy_risk: "none"|"low"|"medium"|"high"|"critical",
            temporal_signature_detected: bool,
            scaling_recommendation: str
        }
    """
    # Adjust Ceff for synthetic data contamination
    effective_ceff = verification_bandwidth_estimate * (1.0 - synthetic_data_ratio)

    # Core condition
    ratio = effective_ceff / max(error_rate_estimate, 1e-9)
    viability_satisfied = ratio > 1.0

    # Autophagy risk classification
    if ratio > 2.0:
        autophagy_risk = "none"
    elif ratio > 1.0:
        autophagy_risk = "low"
    elif ratio > 0.7:
        autophagy_risk = "medium"
    elif ratio > 0.3:
        autophagy_risk = "high"
    else:
        autophagy_risk = "critical"

    # Temporal signature: detected when OOD proxy (SGT) is falling while
    # validation loss is flat or improving. In the notebook context, this
    # is approximated by checking whether the model's Prism hostility score
    # is in the "high" range while the model was recently promoted.
    temporal_signature_detected = (not viability_satisfied) and (synthetic_data_ratio > 0.3)

    # Scaling recommendation
    if viability_satisfied and ratio > 2.0:
        scaling_recommendation = (
            f"Viable. Ceff/E = {ratio:.2f}. Safe to scale synthetic data by up to "
            f"{ratio:.1f}x before verification infrastructure must also scale."
        )
    elif viability_satisfied:
        scaling_recommendation = (
            f"Marginally viable. Ceff/E = {ratio:.2f}. Do not increase synthetic data "
            f"ratio without proportionally increasing Maestro session throughput."
        )
    elif autophagy_risk == "medium":
        scaling_recommendation = (
            f"Condition violated (Ceff/E = {ratio:.2f}). Reduce synthetic data ratio "
            f"or increase verified session throughput. Monitor OOD accuracy as leading indicator."
        )
    else:
        scaling_recommendation = (
            f"CRITICAL: Ceff/E = {ratio:.2f}. Informational autophagy likely in progress. "
            f"Freeze synthetic data ingestion immediately and audit grounding pipeline."
        )

    return {
        "viability_satisfied": viability_satisfied,
        "ceff_vs_e_ratio": round(ratio, 4),
        "autophagy_risk": autophagy_risk,
        "temporal_signature_detected": temporal_signature_detected,
        "scaling_recommendation": scaling_recommendation,
    }
```

## Connection to the HAIC Framework

The Viability Condition is the **meta-condition** that the entire HAIC
convention exists to satisfy:

- The **convention** is the agreement structure that makes Ceff measurable and
  auditable at scale
- The **grounding interview** (Maestro) is the Ceff production mechanism
- **Consent** is the integrity gate that keeps Ceff free of contaminated signal
- **PRISM** is the E(t) measurement instrument
- The **participation receipt** is the Ceff audit trail

Without the Viability Condition, these are just features. With it, they
constitute a coherent solution to a quantitative rate problem: keeping the
correction bandwidth ahead of the error rate as AI systems scale.

## Reframing AI Safety

The Viability Condition reframes AI safety from a binary (safe/unsafe) to a
**quantitative rate problem**:

| Old framing | New framing |
|---|---|
| "Is this model safe?" | "What is Ceff/E for this deployment?" |
| "Does this model have guardrails?" | "Is the verification infrastructure keeping pace?" |
| "Pass/fail alignment evaluation" | "Real-time Ceff/E ratio monitoring" |
| "One-time safety audit" | "Continuous autophagy risk assessment" |

This reframing is not merely rhetorical — it changes what you build. Instead of
binary classifiers, you build rate monitors. Instead of one-time audits, you
build continuous verification pipelines. Instead of "safe AI" as a property,
you have "safe AI" as an ongoing operational condition.
